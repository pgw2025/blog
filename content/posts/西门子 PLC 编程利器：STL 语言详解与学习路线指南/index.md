在工业自动化领域，西门子 S7 系列 PLC 占据着举足轻重的地位。在西门子的编程语言家族中，**LAD（梯形图）** 直观易懂，像电气原理图；**SCL（结构化控制语言）** 类似于高级语言 Pascal/C，适合复杂算法；而 **STL（Statement List，语句表）**，则常被工控工程师称为 **“PLC 的汇编语言”**。

虽然在最新的 S7-1200/1500 控制器中，西门子官方推荐使用 LAD 和 SCL 以配合“优化块访问”特性，但在数以万计的经典 S7-300/400 存量系统、大型复杂指针算法以及老旧项目维护中，STL 依然是一门不可或缺的硬核技能。

---

## 一、 通俗理解：什么是 STL 语言？

如果把 PLC 编程比作指挥一支军队：
* **LAD（梯形图）** 就像是给士兵看**沙盘路线图**（直观，人人看得懂，但不适合微观战术细节）。
* **SCL（高级语言）** 就像是发下**政策红头文件**（逻辑宏观，处理数学计算非常方便）。
* **STL（语句表）** 则像是班长直接对士兵下达的**口令动作**：“立正、向前一步、向左转、把这箱弹药搬过去”（指令直接、执行效率极高、能精准操控 CPU 的每一个寄存器）。

### 为什么还要学习 STL？
1. **老旧项目维护**：大量的早期经典项目（特别是 S7-300/400 时代）的核心算法、逻辑都是用 STL 写的。不掌握 STL，你可能连代码都看不懂。
2. **理解底层运行机制**：STL 让你直接面对 CPU 的寄存器、累加器和状态字，学完 STL 后，你会对 PLC 程序的运行本质有更深刻的认识。
3. **强大的间接寻址（指针）**：在旧版 PLC 中，STL 是实现循环、动态指针寻址、配方管理的利器。

## 二、 STL 的三大底层工作机制

要看懂 STL，必须先明白 PLC 内部有三个特殊的“工作场所”。

### 1. 累加器（ACCU 1 和 ACCU 2）
你可以把累加器想象成两个**“快递中转箱”**。
* 当你执行 `L IW0`（Load，装载）时，PLC 会把 `IW0` 的数据放进 **ACCU 1**。
* 当你再次执行 `L 10` 时，原本在 ACCU 1 里的数据会被自动推入 **ACCU 2**，而新的数据 `10` 进入 **ACCU 1**。
* 执行加法 `+I` 时，PLC 会自动将 ACCU 1 和 ACCU 2 里的数据相加，并将结果保留在 ACCU 1 中。

### 2. 逻辑操作结果（RLO）
**RLO（Result of Logic Operation）** 就像是 PLC 脑子里的一个 **“开关状态暂存器”（只有 0 或 1）**。
* 比如，当你输入 `A I0.0`（逻辑与，And）时，PLC 会看 `I0.0` 是否接通，并把结果存入 RLO。
* 接下来的 `A I0.1` 动作，会把 `I0.1` 的状态与之前 RLO 里的值做逻辑乘积，更新 RLO。
* 最后执行 `= Q0.0`（输出，Assign）时，PLC 就是把最终的 RLO 值写给 `Q0.0`。

### 3. 状态字（Status Word）
这是一个 16 位的寄存器，用来保存 CPU 执行完上一条指令后的状态信息（例如：结果是否为 0？是否溢出？运算结果正负？）。分支跳转指令（如 `JC`，即“若上一结果为 1 则跳转”）就是通过读取状态字来实现的。

## 三、 经典 STL 基础指令拆解

下面通过两个极简单的例子，帮你快速找到读 STL 的感觉。

### 示例 1：点动启停控制逻辑（等同于 LAD 中的线圈输出）
* **LAD 视图**：`I0.0` 与 `Q0.0` 并联，再与 `I0.1` 常闭串联，输出给 `Q0.0`。
* **STL 视图**：
  ```text
  A   I0.0      // 检查启动按钮 I0.0 是否按下
  O   Q0.0      // 或（Or）检查输出自锁 Q0.0 是否接通
  AN  I0.1      // 与非（And Not）常闭停止按钮 I0.1 是否未按下
  =   Q0.0      // 将逻辑运算结果（RLO）输出给线圈 Q0.0
  ```

### 示例 2：模拟量数据转换（数据装载与传送）
* **STL 视图**：
  ```text
  L   PIW 256   // 1. 将外设输入通道 PIW 256 的模拟量数值装载到累加器 ACCU 1
  ITD           // 2. 将 ACCU 1 中的 16位整数(Integer) 转换为 32位双整数(Double Integer)
  DTR           // 3. 将 ACCU 1 中的双整数 转换为 浮点数/实数(Real)
  L   2.7648e+04// 4. 将 27648.0 这个浮点数装载到 ACCU 1，原来的模拟量值被推入 ACCU 2
  /R            // 5. 执行浮点数除法（ACCU 2 / ACCU 1），结果存回 ACCU 1
  T   MD 10     // 6. 将计算出来的浮点数结果传送（Transfer）并存储到 MD 10 变量中
  ```

## 四、 详尽的 STL 语言学习路线大纲

想要系统、彻底地攻克 STL，建议遵循以下由浅入深的步骤：

```text
阶段 1：工控硬核基础 ────> 阶段 2：STL 基础指令 ────> 阶段 3：算术运算与状态字
                                                             │
阶段 6：现代迁移与重构 <─── 阶段 5：指针与间接寻址 <─── 阶段 4：程序控制与FC/FB接口
```

### 阶段 1：工控与微机基础储备
* **目标**：理解计算机底层数据是如何保存和运算的。
* **核心知识点**：
  * **数制系统**：二进制、十进制、十六进制、BCD码。
  * **数据类型**：BOOL（1位）、BYTE（8位）、WORD（16位无符号）、INT（16位有符号）、DWORD（32位无符号）、DINT（32位有符号）、REAL（32位浮点数）。
  * **PLC 内部构造**：理解 CPU 中物理寻址区（I、Q、M、DB、L）的作用。

### 阶段 2：掌握累加器与基础逻辑指令
* **目标**：能够把常见的梯形图无缝翻译成 STL 语句。
* **核心知识点**：
  * **RLO 与 ACCU 1 / ACCU 2** 的演变过程跟踪。
  * **位逻辑指令**：`A` (与), `AN` (与非), `O` (或), `ON` (或非), `X` (异或), `=` (输出), `S` (置位), `R` (复位)。
  * **括号逻辑**：`A(` 和 `)` 的使用，理解复杂串并联网络如何通过括号堆栈实现。
  * **数据装载与传送**：`L` (Load) 与 `T` (Transfer) 在不同数据长度下的执行过程。

### 阶段 3：算术运算、移位与状态字（Status Word）
* **目标**：能够利用 STL 进行复杂的数学计算，并理解运算过程中的状态变化。
* **核心知识点**：
  * **整数数学运算**：`+I`, `-I`, `*I`, `/I`（针对16位），`+D`, `-D`（针对32位）。
  * **实数数学运算**：`+R`, `-R`, `*R`, `/R`。
  * **转换指令**：`ITD`（整转双整）, `DTR`（双整转实数）, `ROUND`（四舍五入舍入为双整）。
  * **状态字详解**：重点学习 `BR` (二进制结果位)、`OV` (溢出位)、`OS` (溢出保持位)、`CC1 / CC0` (比较结果位) 的状态判断。

### 4. 程序流控制与接口变量应用
* **目标**：学习如何在 STL 中编写逻辑跳转、循环，以及如何在 FC/FB 中进行参数传递。
* **核心知识点**：
  * **跳转指令**：`JU` (无条件跳转), `JC` (若 RLO=1 则跳转), `JCN` (若 RLO=0 则跳转)。
  * **循环指令**：`LOOP` 指令的执行原理（如何自动递减 ACCU 1 并实现循环）。
  * **FC / FB 接口编程**：在 STL 状态下，如何正确调用 Temp 临时变量、Stat 静态变量、IN/OUT 接口参数。

### 5. 攻克山峰：指针与间接寻址（STL 的核心威力）
* **目标**：掌握西门子 PLC 最难也是最强大的指针寻址方式。
* **核心知识点**：
  * **16位寻址与32位寻址**的区别。
  * **地址寄存器（AR1、AR2）** 的作用。
  * **区域内间接寻址**（如：`L  M [MD 10]`）。
  * **跨区域间接寻址**（如：`L  DIB [AR1, P#2.0]`，理解 `P#` 指针偏移常量的格式）。
  * **ANY 指针与 POINTER 指针** 的结构剖析（手动拼装数据块指针进行批量数据块复制 `SFC20 BLKMOV`）。

### 6. 现代迁移：STL 阅读重构与 LAD/SCL 的融合
* **目标**：具备将老旧 STL 代码改写、重构为现代化 LAD 或 SCL 代码的能力。
* **核心知识点**：
  * **在博途（TIA Portal）中阅读 STL** 的技巧。
  * **STL 与 SCL 的映射关系**：如何将繁琐的 STL 指针循环语句，优雅地改写为 SCL 中的 `FOR` / `WHILE` 循环。
  * **混编调试**：在 TIA 软件中进行 PLC 仿真（PLCSIM），监控状态字与累加器的动态值。

## 五、 给初学者的复习与实践建议

1. **先在纸上画“堆栈演变”**：
   在学习 L/T 和算术指令时，准备一张纸。写下每一步指令执行后，ACCU 1、ACCU 2 以及 RLO 的值发生了什么变化。画过几次示意图后，你就能在脑海中瞬间反应出指令逻辑。
2. **善用“切换视图”功能**：
   在 STEP 7 或博途软件中，很多逻辑简单的 LAD 块可以直接通过菜单切换成 STL 视图。通过对比同一个网络在 LAD 和 STL 下的不同展现，能够极快地建立直观感觉。
3. **安全提示**：
   由于 STL 可以直接对指针和寄存器进行物理操作，编写错误的指针逻辑（例如越界访问不存在的数据块）可能导致 PLC **立即进入 STOP 状态**。因此，任何 STL 代码的测试与练习，**请务必在仿真软件中（PLCSIM）进行**，切勿直接在运行中的实体设备上测试。
