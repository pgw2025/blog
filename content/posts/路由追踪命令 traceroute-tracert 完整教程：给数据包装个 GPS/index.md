# 路由追踪命令完整教程：给数据包装个 GPS

当你发现访问某个网站很慢，或者网络连接在中途断了，你怎么知道问题出在哪里？是在你家路由器？在运营商骨干网？还是在目标服务器的门口？

这时候，路由追踪工具（Windows 下的 `tracert` 和 Linux 下的 `traceroute`）就是你的“探路先锋”。

---

## 一、什么是路由追踪？

**路由追踪**（Route Tracing）是一种通过发送特殊的数据包，来记录从你的电脑到目标服务器之间经过了哪些“车站”（路由器）的工具。

**形象比喻：** 就像你从北京开车去上海，路由追踪会告诉你你经过了济南、南京等哪些服务区，并且在每个服务区停留了多久。如果车堵在了南京，你一眼就能看出来。

---

## 二、Windows 平台：tracert 命令

在 Windows 的 CMD 或 PowerShell 中，我们使用 `tracert`。

### 1. 基本用法
```cmd
tracert www.baidu.com
```

### 2. 常用参数详解
- **`-d`**：**不解析主机名**。默认情况下，`tracert` 会尝试把每个 IP 转换成域名，这很慢。加上 `-d` 可以大幅提升追踪速度。
- **`-h <max_hops>`**：**指定最大跳数**。默认是 30 跳。如果目标很远，你可以调大它（如 `-h 50`）。
- **`-w <timeout>`**：**设置超时时间**（单位毫秒）。如果某个节点响应慢，你可以手动增加等待时间。
- **`-4` 或 `-6`**：强制使用 IPv4 或 IPv6。

**推荐用法：**
```cmd
tracert -d www.google.com
```

---

## 三、Linux/macOS 平台：traceroute 命令

在 Linux 系统中，功能更强大，通常使用 `traceroute`。

### 1. 基本用法
```bash
traceroute www.baidu.com
```

### 2. 常用参数详解
- **`-n`**：**不进行 DNS 解析**。直接显示 IP 地址，速度最快（对应 Windows 的 `-d`）。
- **`-m <max_hops>`**：设置最大跳数（对应 Windows 的 `-h`）。
- **`-w <seconds>`**：设置超时时间（**注意：Linux 下单位通常是秒**）。
- **`-q <n>`**：设置每一跳发送的探测包数量（默认是 3 个）。
- **`-T`**：**使用 TCP SYN 包**。这是最实用的参数！因为很多防火墙会拦截默认的 UDP 或 ICMP 包，导致追踪时出现一堆星号 `*`，用 `-T` 可以绕过。
- **`-I`**：使用 ICMP Echo 包（类似于 Windows `tracert` 的默认方式）。

**推荐用法：**
```bash
# 使用 TCP 协议追踪 80 端口，不解析域名
traceroute -n -T -p 80 www.example.com
```

## 四、核心原理：它是怎么工作的？

为什么 `traceroute` 能知道中途路由器的 IP？这利用了 IP 协议里的 **TTL (Time To Live)** 机制。

1.  **第一步**：你的电脑发一个数据包，把 **TTL 设为 1**。
2.  **第二步**：第一个路由器收到包，把 TTL 减 1 变成 0，于是包“死”了。路由器会发回一个“死亡证明”（ICMP 时间超时包），里面包含它的 IP。
3.  **第三步**：你的电脑再发一个包，**TTL 设为 2**。
4.  **第四步**：经过第一个路由器（变 1），到达第二个路由器（变 0），第二个路由器发回身份信息。
5.  **以此类推**，直到包到达终点。

## 五、如何看懂追踪结果？

运行命令后，你会看到类似下面的列表：

```text
Tracing route to www.example.com [93.184.216.34] over a maximum of 30 hops:
  1    <1 ms    <1 ms    <1 ms  192.168.1.1
  2    15 ms    12 ms    13 ms  10.26.0.1
  3     *        *        *     Request timed out.
  4    45 ms    48 ms    46 ms  202.97.34.122
```

### 1. 三列时间（ms）
每一跳会发送 3 个包，显示 3 个耗时。
- **时间短且平稳**：网络良好。
- **时间突然激增**：从这一跳开始，网络出现了拥塞。

### 2. 出现星号 `*`
如果某一行全是 `*`，代表 **Request timed out**（请求超时）。
- **中间某一跳是星号**：通常是该路由器开启了防火墙，禁止响应探测包。只要后面几跳能通，通常没问题。
- **后面全是星号**：代表网络在中途彻底断了，或者目标服务器禁止被追踪。

### 3. 第一跳和第二跳
- **第 1 跳**：通常是你的**无线路由器**（如 192.168.x.x）。如果这一跳延迟很高，说明你家 WiFi 信号不好。
- **第 2 跳**：通常是你的**运营商（ISP）网关**。如果这一跳就 100ms+，说明你家到机房的线路有问题。

## 六、命令参数速查表

| 功能 | Windows (`tracert`) | Linux (`traceroute`) | 建议 |
| :--- | :--- | :--- | :--- |
| **不解析域名** | `-d` | `-n` | **必带**，速度快 10 倍 |
| **最大跳数** | `-h` | `-m` | 默认 30 够用 |
| **设置超时** | `-w` (ms) | `-w` (s) | 网络极差时调大 |
| **指定协议** | 默认 ICMP | `-T`(TCP) / `-I`(ICMP) | 遇到星号改用 `-T` |
| **发送包数** | 不支持(固定3) | `-q` | 默认 3 够用 |

---

## 七、总结：排障顺口溜

**一跳高，看路由（家里的 WiFi 或路由器）；**
**二跳高，找电信（运营商线路故障）；**
**中间星，防墙拦（路由器防火墙拦截）；**
**后面星，链路断（网络彻底不通）。**

路由追踪是网络工程师的“法眼”，学会看这几行简单的输出，你就能比普通用户更早定位网络问题的根源！
