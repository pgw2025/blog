# npm 全攻略：前端开发的“百宝箱”

如果你在做前端开发，那么 **npm (Node Package Manager)** 绝对是你每天都要打交道的“老朋友”。它不仅是 Node.js 的包管理器，更是目前全球最大的开源代码生态系统。

通俗点说，npm 就像是一个超级巨大的“零件仓库”。你想给网页加个轮播图？仓库里有！想处理复杂的日期格式？仓库里也有！你只需要一个命令，它就能把零件搬到你的项目里。

---

## 1. 基础入门：初始化你的项目

在开始搬零件之前，你得先有个“仓库清单”。

### 🛠️ npm init —— “开张大吉”
这个命令会在你的项目根目录下生成一个 `package.json` 文件。
*   **常用参数：**
    *   `-y, --yes`: 所有的询问都选“Yes”。适合想快速开搞，不想一个一个回答项目名称、版本号的人。
        *   例：`npm init -y`

---

## 2. 核心命令：搬运零件（安装依赖）

### 📥 npm install (简写: npm i) —— “进货”
这是用得最频繁的命令。

*   **常用用法：**
    *   `npm install`: 根据 `package.json` 里的清单，一次性把所有零件都买回来（安装所有依赖）。
    *   `npm install <pkg>`: 安装某个具体的包。
*   **核心参数：**
    *   `-g, --global`: **全局安装**。零件不放项目里，而是放在系统里，通常用于命令行工具（如 `hugo` 或 `nodemon`）。
    *   `-S, --save`: **生产依赖**（默认）。零件是房子的一部分，上线也得带着。（记录在 `dependencies` 中）
    *   `-D, --save-dev`: **开发依赖**。零件只是脚手架，盖完房子就扔，上线不需要。（记录在 `devDependencies` 中）
    *   `-E, --save-exact`: **精准安装**。固定版本号，防止以后自动更新导致代码跑不通。

---

## 3. 维护与清理：整理仓库

### 🗑️ npm uninstall (简写: npm un) —— “退货”
不想要的零件，我们要把它删干净。
*   例：`npm uninstall lodash`

### 🔄 npm update (简写: npm up) —— “换新件”
检查并更新你的零件到最新的兼容版本。

### 📋 npm list (简写: npm ls) —— “查库存”
看看你到底装了多少零件，以及它们之间的套娃关系。
*   **参数：** `--depth=0` 只看第一层，不然会看得你眼花缭乱。

---

## 4. 脚本执行：让项目“动起来”

### 🏃 npm run <script> —— “开机”
在 `package.json` 的 `scripts` 字段里，你可以定义很多快捷键。
*   **常见用法：**
    *   `npm run dev`: 启动开发服务器。
    *   `npm run build`: 打包项目。
*   **注意：** `start` 和 `test` 是两个特殊命令，可以省略 `run`，直接写 `npm start`。

---

## 5. 进阶黑科技：你可能不知道的

### 🔍 npm view <pkg> versions —— “看货架”
想看看这个包都有过哪些版本？一个命令搞定。

### 🧼 npm cache clean --force —— “大扫除”
如果安装总是报错，或者提示文件损坏，试试清理一下缓存。

### 🧪 npx —— “只租不买”
这是 npm 5.2 之后自带的利器。如果你想临时用一下某个工具（比如 `create-react-app`），但不想全局安装占空间，就用 `npx`。
*   例：`npx create-react-app my-app`

---

## 6. 常见疑惑 Q&A

**Q: `package.json` 和 `package-lock.json` 有啥区别？**
A: `package.json` 是你的“购物意向清单”（比如：我要 4.x 版本的零件）；`package-lock.json` 是你的“购物小票”，精准记录了上次买回来的到底是 4.1.2 还是 4.1.5，确保下次买的一模一样。

**Q: 为什么我安装东西特别慢？**
A: 因为 npm 的主仓库在国外。你可以换成国内的镜像（比如淘宝镜像）：
`npm config set registry https://registry.npmmirror.com`
