# Git 全指令史诗级通关指南

如果把写代码比作写小说，那么 **Git** 就是你的“全能后悔药”和“时光机”。它可以记录你每一个章节的改动，让你随时回到过去，或者在不破坏主线剧情的情况下尝试多种结局。

今天，我们不只是列出指令，而是要把 Git 的“全家桶”及其常用参数一次性讲透！

---

## 1. 建立根据地：初始化与克隆

### 🏗️ git init —— “这里归我管了”
在当前文件夹创建一个隐藏的 `.git` 文件夹，从此这个目录的一举一动都会被 Git 记录。

### 🚚 git clone <url> —— “搬运过来”
把远程仓库（比如 GitHub 上的项目）完整地复制到你本地。
*   **常用参数：**
    *   `-b <branch>`: 只克隆某个特定的分支。
    *   `--depth 1`: 浅克隆。只拉取最近一次提交，适合超大型项目，速度飞快。

---

## 2. 每日修仙：基础工作流

### 📝 git add —— “打包进发货区”
把你的改动从“工作区”挪到“暂存区（Stage）”。
*   **常用参数：**
    *   `git add .`: 把当前目录下所有改动都加上。
    *   `git add <file>`: 只加某个特定文件。
    *   `-p`: 交互式添加。它会问你每一块改动要不要加，适合一次性改了很多地方但想分批提交的情况。

### 💾 git commit —— “按下保存键”
把暂存区的东西正式存入版本库，生成一个唯一的“哈希值”（ID）。
*   **常用参数：**
    *   `-m "message"`: 必须带的消息，说明你这次改了啥。
    *   `-a`: 偷懒专用。把已跟踪文件的修改直接 add 并 commit（对新文件无效）。
    *   `--amend`: **“打脸”专用**。发现刚才的 commit 消息写错了或者少传了文件？用这个可以覆盖上一次提交。

### 🔍 git status & diff —— “找不同”
*   `git status`: 看看谁改了、谁没加、谁在暂存区。
*   `git diff`: 详细看代码改了哪一行。
    *   `--cached`: 查看暂存区和最后一次提交的区别。

---

## 3. 开启平行宇宙：分支管理

### 🌿 git branch —— “看有多少个结局”
*   `git branch -a`: 查看本地和远程的所有分支。
*   `git branch <name>`: 创建新分支。
*   `-d <name>`: 删除分支。

### 🚀 git checkout / switch —— “穿越时空”
*   `git checkout -b <name>`: 创建并切换到新分支。
*   `git switch <name>`: 新版 Git 推荐的切换指令，语义更清晰。

### 🤝 git merge —— “合二为一”
把 A 分支的改动合并到当前分支。
*   **常用参数：**
    *   `--no-ff`: 强行留下合并痕迹（生成一个合并提交），方便以后回溯。

---

## 4. 远征海外：远程仓库

### 📡 git remote —— “建立连接”
*   `git remote add origin <url>`: 关联一个远程仓库。

### ⬆️ git push —— “发货”
*   `-u origin master`: 第一次推送到时候用，以后直接 `git push` 就行。
*   `-f`: **“核武器”——强制推送**。除非你确定只有你一个人在用这个分支，否则别乱动！

### ⬇️ git pull & fetch —— “进货”
*   `git fetch`: 只把远程的改动拉下来，不合并。
*   `git pull`: 拉下来并自动合并（等于 fetch + merge）。
    *   `--rebase`: 推荐用法。让你的提交历史变成一条直线，而不是扭来扭去的麻花。

---

## 5. 后悔药专区：撤销操作

### ⏪ git reset —— “时光倒流”
*   `--soft`: 撤销 commit，但改动的代码还在，且留在暂存区。
*   `--mixed`:（默认）撤销 commit 和暂存区，代码还在工作区。
*   `--hard`: **彻底抹除**。回到过去某个状态，之后的改动全部灰飞烟灭。

### 🔄 git revert —— “拨乱反正”
生成一个新的提交来“抵消”之前的某个提交。比起 `reset`，它更安全，因为它不改写历史。

### 📦 git stash —— “临时小推车”
代码写一半，突然要去改 Bug 怎么办？
*   `git stash`: 把现在的改动先藏起来。
*   `git stash pop`: Bug 改完了，把藏起来的代码再拿出来。

---

## 6. 查看历史：翻旧账

### 📜 git log —— “翻日记”
*   `--oneline`: 一行显示一条，清爽。
*   `--graph`: 字符画展示分支合并图，非常硬核。
*   `--all`: 看所有分支的历史。

---

## 总结：你的 Git 快捷口诀

1.  **开工前**：`git pull --rebase`（保持最新）。
2.  **改完后**：`git add .` -> `git commit -m "..."`。
3.  **要发布**：`git push`。
4.  **要尝试**：`git checkout -b feat-new`。
5.  **搞砸了**：不要慌，先 `git status` 看看在哪。

**Git 不只是工具，它是一种工作流思维。掌握了这些，你就是团队里的“代码守护神”！**
