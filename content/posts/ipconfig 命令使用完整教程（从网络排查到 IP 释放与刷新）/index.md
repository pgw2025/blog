# ipconfig 命令使用完整教程

当你的电脑连不上网，或者你想知道自己电脑的内网 IP 地址时，`ipconfig` 绝对是你第一个要敲下的命令。它是 Windows 系统中查看和管理网络配置最常用的工具。

本文将带你全面了解 `ipconfig` 的各种用法，让你在遇到网络问题时不再抓瞎。

---

## 一、什么是 ipconfig？

`ipconfig` 是 **Internet Protocol Configuration**（网际协议配置）的缩写。

**简单来说：** 它就像是电脑网络适配器的“身份证读取器”。通过它，你可以看到电脑当前连接的是哪个网络、分到了什么 IP 地址，以及你的网关（路由器）在哪里。

---

## 二、基础入门：查看你的网络信息

### 1. 快速查看 IP 地址
直接在命令提示符（CMD）里输入 `ipconfig`。
```cmd
ipconfig
```
**你能看到什么：**
- **IPv4 地址**：你电脑在局域网里的“门牌号”（如 `192.168.1.5`）。
- **子网掩码**：通常是 `255.255.255.0`。
- **默认网关**：你路由器的地址（如 `192.168.1.1`）。如果你能 ping 通网关但上不了网，那通常是宽带出问题了。

### 2. 查看详细信息 (`/all`)
如果你需要查看更深层的信息，比如 **MAC 地址**（物理地址）或 **DNS 服务器**。
```cmd
ipconfig /all
```
**进阶信息：**
- **物理地址 (MAC)**：你网卡的唯一硬件 ID。
- **DHCP 已启用**：如果是“是”，说明 IP 是路由器自动分配的。
- **DNS 服务器**：负责把网址转换成 IP 的服务器。

---

## 三、解决断网：释放与重新获取 IP

有时候电脑显示“受限连接”或“无法获取 IP 地址”，这时可以通过“先释放、再重新获取”的策略来解决。

### 1. 释放当前 IP (`/release`)
告诉路由器：“这个 IP 我不要了，你收回去吧。” 运行后你会暂时断网。
```cmd
ipconfig /release
```

### 2. 重新获取 IP (`/renew`)
告诉路由器：“请重新给我分配一个可用的 IP 地址。” 
```cmd
ipconfig /renew
```
**实战场景：** 当你的电脑和别人 IP 冲突，或者路由器死机重启后电脑没反应，用这两条命令通常能修好。

---

## 四、DNS 故障排查：刷新 DNS 缓存

如果你发现能上 QQ（不需要 DNS），但打不开网页（需要 DNS），那多半是 DNS 缓存出问题了。

### 1. 刷新 DNS 缓存 (`/flushdns`)
清理电脑里存储的旧网址记录，强制电脑去询问最新的解析结果。
```cmd
ipconfig /flushdns
```
**提示：** 当网站换了服务器导致你打不开网页时，这招非常管用。

### 2. 查看当前 DNS 缓存 (`/displaydns`)
看看你电脑最近都访问过哪些域名及其对应的 IP。
```cmd
ipconfig /displaydns
```

---

## 五、进阶操作：注册与通告

### 1. 重新注册 DNS (`/registerdns`)
如果你的电脑是加入公司域（Domain）的，这条命令可以手动触发电脑在 DNS 服务器上更新自己的记录。
```cmd
ipconfig /registerdns
```

---

## 六、常用参数速查表

| 参数 | 功能说明 | 适用场景 |
| :--- | :--- | :--- |
| `ipconfig` | 查看基础 IP、掩码和网关 | 快速看一眼自己的 IP |
| `ipconfig /all` | 查看详细配置（MAC, DNS, DHCP） | 需要查网卡物理地址时 |
| `ipconfig /release` | 释放当前的 IP 地址 | 准备手动断开当前连接 |
| `ipconfig /renew` | 重新向服务器申请 IP | 解决“无法连接网络”或 IP 冲突 |
| `ipconfig /flushdns` | 清除 DNS 解析缓存 | 网页打不开，提示 DNS 解析错误 |
| `ipconfig /displaydns` | 显示当前的 DNS 缓存内容 | 检查电脑最近访问了哪些域名 |

---

## 七、总结：排查网络的“三板斧”

当你遇到 Windows 联网问题时，建议按以下顺序操作：
1. **看**：`ipconfig` 确认是否有 IP，网关是否对。
2. **刷**：`ipconfig /flushdns` 排除域名解析干扰。
3. **换**：`ipconfig /release` 然后 `ipconfig /renew` 重置连接。

如果这三步做完还没好，那可能该检查网线或者重启路由器了！
