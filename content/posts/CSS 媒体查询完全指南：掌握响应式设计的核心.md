# CSS 媒体查询完全指南：掌握响应式设计的核心

在移动互联网时代，你的网页可能会在 5 英寸的手机上运行，也可能会在 32 英寸的 4K 显示器上运行。**媒体查询 (Media Queries)** 就是网页的“眼镜”，它让网页能够感知设备的特征（如屏幕宽度），并应用不同的样式。

---

## 一、 什么是媒体查询？

通俗来说，媒体查询就像是一句条件判断：
> “**如果** 屏幕宽度大于 768 像素，**那么** 就把背景变成蓝色。”

### 基本语法
```css
@media 媒体类型 and (媒体特性) {
    /* 这里的样式只在条件满足时生效 */
    body {
        background-color: lightblue;
    }
}
```
- **媒体类型**：常用 `screen`（屏幕）。
- **媒体特性**：最常用的是 `width`（宽度）。

---

## 二、 核心逻辑：min-width vs max-width

这是初学者最容易混淆的地方。其实只要记住这六个字：**“起步价”与“封顶价”**。

### 1. min-width (最小宽度) —— “向上看”
- **含义**：屏幕宽度**至少**为多少时生效。
- **逻辑**：当屏幕变宽，超过这个值，样式生效。
- **关联策略**：**移动端优先 (Mobile First)**。

### 2. max-width (最大宽度) —— “向下看”
- **含义**：屏幕宽度**至多**为多少时生效。
- **逻辑**：当屏幕变窄，低于这个值，样式生效。
- **关联策略**：**桌面端优先 (Desktop First)**。

---

## 三、 两大布局策略：你选哪一个？

### 1. 移动端优先 (Mobile-First) —— 【强烈推荐】
这是现代前端开发的标准做法。

- **思路**：先写手机端的样式（最基础、最简单的样式），然后通过 `min-width` 随着屏幕变大逐渐增加复杂度。
- **代码示例**：
```css
/* 默认样式 (手机端) */
.container {
    width: 100%;
}

/* 平板端 (大于 768px) */
@media screen and (min-width: 768px) {
    .container {
        width: 750px;
    }
}

/* 电脑端 (大于 1200px) */
@media screen and (min-width: 1200px) {
    .container {
        width: 1140px;
    }
}
```
- **优点**：代码逻辑清晰，从简单到复杂，且对移动端性能更友好。

---

### 2. 桌面端优先 (Desktop-First)
这是传统做法，或者是对老网站进行响应式改造时常用的方法。

- **思路**：先写电脑端的复杂样式，然后通过 `max-width` 随着屏幕变小逐渐简化或隐藏元素。
- **代码示例**：
```css
/* 默认样式 (电脑端) */
.sidebar {
    display: block;
    width: 300px;
}

/* 手机端 (小于 768px) */
@media screen and (max-width: 768px) {
    .sidebar {
        display: none; /* 屏幕太小了，隐藏侧边栏 */
    }
}
```

---

## 四、 常用断点 (Breakpoints) 推荐

虽然不同项目有不同需求，但以下是业界公认的“黄金断点”：

- **超小屏幕 (手机)**: `< 576px`
- **小屏幕 (平板竖屏)**: `≥ 576px`
- **中等屏幕 (平板横屏)**: `≥ 768px`
- **大屏幕 (笔记本/小显示器)**: `≥ 992px`
- **超大屏幕 (大显示器)**: `≥ 1200px`

---

## 五、 避坑指南：顺序非常重要！

当你使用多个媒体查询时，CSS 的“就近原则”会生效。

1. **如果你用 `min-width` (移动端优先)**：请务必按**从小到大**的顺序写（如 768 -> 992 -> 1200）。
2. **如果你用 `max-width` (桌面端优先)**：请务必按**从大到小**的顺序写（如 1200 -> 992 -> 768）。

如果顺序写反了，大的范围会覆盖小的范围，导致你的媒体查询失效。

---

## 六、 总结图表

| 策略 | 常用特性 | 书写顺序 | 核心思想 |
| :--- | :--- | :--- | :--- |
| **移动端优先** | `min-width` | **从小到大** | 简单 -> 复杂 (加法) |
| **桌面端优先** | `max-width` | **从大到小** | 复杂 -> 简单 (减法) |

---

## 七、 结语

媒体查询不是要把所有 CSS 重新写一遍，而是**针对性地修改**那些随屏幕变化的属性（如 `width`, `display`, `font-size`）。掌握了 Mobile-First 策略和 `min-width` 的用法，你就拿到了通往响应式设计大门的钥匙。
