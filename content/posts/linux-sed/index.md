在 Linux 运维和开发中，经常需要对配置文件、日志文件进行自动化修改或提取。如果使用传统的文本编辑器（如 `vi`/`vim`），必须打开文件、定位、修改、保存，这在编写自动化脚本或处理海量日志时几乎无法实现。

**`sed`（Stream Editor，流编辑器）** 专门用于解决这种场景。它是一个非交互式的面向字符流的编辑器，能根据设定的规则直接对管道输出或文件内容进行实时编辑。

---

## 一、 sed 的工作原理

理解 `sed` 的工作模式有助于写出逻辑清晰的过滤命令。`sed` 遵循以下循环步骤：

1. **读取**：从输入流（文件或管道）中读取一行，存入临时的缓冲区，称为**模式空间（Pattern Space）**。
2. **执行**：在模式空间中，运行指定的 `sed` 编辑命令。如果有多条命令，按顺序依次执行。
3. **输出**：默认情况下，将模式空间处理后的内容输出到标准输出（终端），然后清空模式空间。
4. **循环**：重复上述过程，直到文件末尾。

> **核心特性**：因为默认操作是在“模式空间”中进行的，**`sed` 默认不会修改源文件的内容**，除非显式指定了 `-i` 参数。

## 二、 命令格式与常用参数

`sed` 的基本命令结构如下：
```bash
sed [参数] '编辑命令' [目标文件]
```

### 1. 常用参数说明

| 参数 | 英文全称 | 物理意义与价值 |
| :---: | :--- | :--- |
| **`-n`** | quiet / silent | **取消默认输出**。默认情况下 `sed` 会打印每一行。配合 `-n` 和 `p` 动作，可以只打印被编辑命令匹配到的行。 |
| **`-i`** | in-place | **直接修改源文件内容**。不输出到标准输出。建议配合备份后缀使用（如 `-i.bak`）。 |
| **`-e`** | expression | 多点编辑。允许在一条命令中执行多个编辑动作。 |
| **`-E`** / **`-r`** | extended-regexp | **启用延伸型正则表达式**（Extended Regular Expression, ERE）。支持 `+`、`?`、`\|`、`()` 等符号，无需繁琐的转义。 |

## 三、 核心编辑动作

在 `'编辑命令'` 中，最核心的是定位（地址范围）加动作。常用的编辑动作如下：

### 1. `s` (substitute) —— 替换（最常用）
语法结构：`s/源模式/替换内容/标记`

* **常用标记（Flags）**：
  * **`g`**：全局替换（Global）。默认只替换每一行中第一次出现的匹配项，加 `g` 替换整行所有匹配项。
  * **`p`**：配合 `sed -n` 使用，只打印发生替换的行。
  * **`i` / `I`**：忽略大小写。
* **分界符替换**：
  默认使用 `/` 作为分界符。如果替换的内容中包含大量斜杠（如 URL 路径 `http://localhost/api`），可以将分界符换成其他字符（如 `#` 或 `@`），避免频繁转义：
  ```bash
  sed 's#http://localhost#https://api.example.com#g' file.txt
  ```

### 2. `d` (delete) —— 删除
用于删除符合条件的行。

### 3. `p` (print) —— 打印
用于输出匹配的行（通常与 `-n` 参数配合）。

### 4. `a` (append) / `i` (insert) —— 新增
* **`a`**：在指定行**之后**追加一行或多行。
* **`i`**：在指定行**之前**插入一行或多行。

### 5. `c` (change) —— 整行替换
将匹配到的整行内容直接替换为新的文本。

## 四、 如何指定地址范围（定位）

如果不指定地址，`sed` 会对文本的**每一行**执行编辑动作。通过指定地址，可以精确控制作用范围：

### 1. 按行号定位
* `3d`：删除第 3 行。
* `1,5p`：打印 1 到 5 行。
* `5,$d`：从第 5 行一直删除到最后一行（`$` 代表最后一行）。
* `1~2p`：从第 1 行开始，每隔 2 行打印一次（即打印所有奇数行）。

### 2. 按正则表达式定位
* `/pattern/d`：删除所有包含 `pattern` 的行。
* `/^#/d`：删除所有以 `#` 开头的行（常用于过滤注释）。
* `/^$/d`：删除所有空行。

### 3. 混合定位
* `1,/pattern/d`：从第 1 行删除到第一个匹配 `pattern` 的行。

## 五、 实战场景与命令模板

### 场景一：替换文件中的敏感词（直接修改文件，带备份）
直接修改文件是危险操作。为了安全，建议使用 `-i.bak` 参数。系统会先生成一个 `.bak` 的备份文件，再修改原文件：

```bash
# 将 db.conf 中的 localhost 替换为 192.168.1.100，并自动生成 db.conf.bak
sed -i.bak 's/localhost/192.168.1.100/g' db.conf
```

### 场景二：注释/取消注释配置文件某一行
在运维中经常需要临时修改服务的配置。
```bash
# 1. 注释掉：将以 "PORT=80" 开头的行前面加上 "#"
sed -i 's/^PORT=80/#PORT=80/' config.ini

# 更优雅的方式（使用正则表达式匹配并保留原字符）：
sed -i 's/^\(PORT=80\)/#\1/' config.ini

# 2. 取消注释：将 "#DEBUG=true" 前面的 "#" 去掉
sed -i 's/^#DEBUG=true/DEBUG=true/' config.ini
```

### 场景三：提取特定时间段的日志
假设日志格式每行开头为 `2026-06-27 10:00:00`，我们需要提取 10:00:00 到 10:05:00 之间的所有日志：

```bash
# -n 配合 p 动作，只打印匹配范围内的行
sed -n '/2026-06-27 10:00:00/,/2026-06-27 10:05:00/p' access.log
```

### 场景四：清理配置文件中的注释行和空行
在查看复杂的开源项目配置文件时（如 `redis.conf`、`nginx.conf`），大量注释和空行会干扰阅读。可以用 `sed` 进行净化：

```bash
# -e 执行多个编辑动作：删除以 # 开头的行，再删除空行
sed -e '/^#/d' -e '/^$/d' redis.conf

# 使用正则表达式联合简化版（需要启用 -E 参数）
sed -E '/^(#|$)/d' redis.conf
```

### 场景五：在指定行后插入新内容
向 `nginx.conf` 的第 10 行后添加一条配置参数：

```bash
sed -i '10a\    client_max_body_size 20m;' nginx.conf
```

### 场景六：使用后向引用（Backreference）进行文本重组
后向引用是 `sed` 的高级功能。通过 `()` 分组匹配，在替换时用 `\1`、`\2` 代表分组中的内容：

```bash
# 假设有一个文本格式为 "IP: 192.168.1.1"
echo "IP: 192.168.1.1" | sed -E 's/IP: ([0-9.]+)/\1/'
# 输出：192.168.1.1
```

## 六、 运维最佳安全实践建议

1. **先预览，后修改**：
   在执行任何带 `-i` 参数的修改命令之前，**务必先去掉 `-i` 进行测试运行**。确认终端输出符合预期后，再加 `-i` 写入文件。
2. **善用备份**：
   在编写自动化部署脚本时，修改关键系统配置文件（如 `/etc/ssh/sshd_config`）必须使用 `-i.bak` 或提前进行 `cp` 备份。
