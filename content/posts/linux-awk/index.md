在 Linux 系统中，如果说 `grep` 是快速定位文本的“探测器”，那么 `awk` 就是处理结构化文本的“瑞士军刀”。

`awk` 不仅仅是一个命令，它其实是一门功能完整的**文本处理编程语言**（以其发明者 Aho, Weinberger 和 Kernighan 的首字母命名）。在日常工作中，它最常被用来处理“行列结构”的数据，比如日志、表格、配置文件等，堪称命令行里的“轻量级 Excel”。

本文将用通俗易懂的语言，由浅入深地为您讲解 `awk` 的工作原理、核心参数、内置变量以及高频实用场景。

---

### 一、 `awk` 的工作原理与基本语法

#### 1. 核心工作流程
`awk` 的工作方式非常简单且有规律：
1. **按行读取**：一次只读取文本中的一行。
2. **切分字段**：默认以**空格**或**制表符（Tab）**为分隔符，将这一行内容切分成若干个“列”（在 `awk` 中称为“字段”）。
3. **匹配与处理**：判断这一行是否符合你指定的条件（Pattern），如果符合，就执行你指定的动作（Action）。
4. **循环往复**：重复上述步骤，直到文件读取完毕。

#### 2. 基本语法格式
```bash
awk [参数] '条件 { 动作 }' 文件名
```
*   **条件（Pattern）**：用来决定“在哪一行起作用”（比如：第三行、或者包含 "Error" 的行）。如果不写，默认对所有行起作用。
*   **动作（Action）**：决定“具体要做什么”（最常用的是 `print` 打印输出）。必须用花括号 `{}` 包起来。

#### 3. 字段（列）的表示方法
在 `awk` 中，切分出来的列用 `$` 符号配合数字来表示：
*   `$0`：代表**整行**文本。
*   `$1`：代表第一列。
*   `$2`：代表第二列。
*   `$N`：代表第 N 列。

##### 准备一个测试文件 `score.txt`：
```text
Tom    90  85  95
Jerry  88  92  80
Spike  60  55  70
```

*   **最简单的例子**：打印每一行的名字（第1列）和数学成绩（第2列）。
    ```bash
    awk '{print $1, $2}' score.txt
    ```
    **输出**：
    ```text
    Tom 90
    Jerry 88
    Spike 60
    ```

### 二、 `awk` 的核心参数（Options）

为了适应不同的文本格式，`awk` 提供了几个关键的命令行参数。

#### 1. `-F` (Field Separator) —— 指定输入分隔符
默认情况下，`awk` 认为空格或 Tab 是列的分隔符。但很多系统文件（如 `/etc/passwd`）是用冒号 `:` 或逗号 `,` 分隔的。这时就需要用 `-F` 明确指定。

*   **示例**：提取系统用户文件 `/etc/passwd` 中的用户名（第1列）和主目录（第6列）。
    ```bash
    # /etc/passwd 的格式类似：root:x:0:0:root:/root:/bin/bash
    awk -F ":" '{print $1, $6}' /etc/passwd
    ```
    *(也可以写成 `-F:` )*

#### 2. `-v` (Variable assignment) —— 引入外部变量
如果你想在 `awk` 内部使用 Shell 脚本中的变量，可以用 `-v` 来传递。

*   **示例**：将外部 Shell 变量 `threshold` 传入 `awk` 中使用。
    ```bash
    threshold=80
    awk -v limit=$threshold '$2 > limit {print $1, $2}' score.txt
    ```
    *(这条命令会筛选出第二列成绩大于 80 的人)*

#### 3. `-f` (Program file) —— 调用 awk 脚本文件
当你的 `awk` 处理逻辑非常复杂，写在单行命令里不方便阅读时，可以把代码写进一个文件里，然后用 `-f` 调用。

*   **示例**：
    ```bash
    # 将逻辑写入 script.awk 文件中
    awk -f script.awk score.txt
    ```

### 三、 `awk` 的内置变量（重点）

`awk` 内置了许多非常有用的全局变量，熟练使用它们可以省去很多复杂的计算逻辑。

| 变量名 | 全称意义 | 实用解释 |
| :--- | :--- | :--- |
| **`NF`** | **N**umber of **F**ields | 当前行**一共有多少列**。 |
| **`NR`** | **N**umber of **R**ecords| 当前处理的是**第几行**（行号）。 |
| **`FS`** | **F**ield **S**eparator | 输入字段分隔符（等同于 `-F` 参数）。 |
| **`OFS`** | **O**utput **F**ield **S**eparator | 输出字段分隔符（控制打印出的列之间用什么连接）。 |
| **`FNR`** | **F**ile **N**umber of **R**ecords | 当前文件中的行号（处理多个文件时，每个文件重新从1计数）。 |

#### 内置变量实例演示：

##### 1. 使用 `NF` 获取最后一列
因为每一行的列数可能不一样，`$NF` 永远代表**最后一列**，`$(NF-1)` 代表倒数第二列。
*   **示例**：
    ```bash
    awk '{print $NF}' score.txt
    ```
    *(输出每个人最后一门课的成绩)*

##### 2. 使用 `NR` 打印行号或过滤特定行
*   **示例 A**：给输出内容加上行号。
    ```bash
    awk '{print NR, $0}' score.txt
    ```
*   **示例 B**：只看第二行。
    ```bash
    awk 'NR==2 {print $0}' score.txt
    ```

##### 3. 使用 `OFS` 修改输出连接符
默认情况下，`print $1, $2` 输出时，两列之间是用空格隔开的。你可以通过 `OFS` 修改这个连接符。
*   **示例**：把输出的列用“横线”连接。
    ```bash
    awk -v OFS="---" '{print $1, $2}' score.txt
    ```
    **输出**：
    ```text
    Tom---90
    Jerry---88
    Spike---60
    ```

### 四、 条件判断与模式匹配（Pattern）

`awk` 支持非常丰富的过滤条件。只有满足条件的行，才会执行后面的花括号动作。

#### 1. 数值比较
常用的操作符有：`>`、`<`、`>=`、`<=`、`==`、`!=`。
*   **示例**：找出第二列成绩大于或等于 80 的所有行。
    ```bash
    awk '$2 >= 80 {print $0}' score.txt
    ```

#### 2. 正则表达式匹配
你可以使用波浪号 `~` 表示“匹配”，`!~` 表示“不匹配”。正则内容写在双斜杠 `//` 之间。
*   **示例 A**：找出名字以 "J" 开头的行。
    ```bash
    awk '$1 ~ /^J/ {print $0}' score.txt
    ```
*   **示例 B**：整行文本中包含 "90" 的行。
    ```bash
    awk '/90/ {print $0}' score.txt
    ```

#### 3. 逻辑组合
支持与 `&&`、或 `||`、非 `!`。
*   **示例**：找出第二列大于 80 **且** 第三列也大于 80 的行。
    ```bash
    awk '$2 > 80 && $3 > 80 {print $1}' score.txt
    ```

### 五、 特殊的 BEGIN 和 END 模块

在 `awk` 中，有两个非常特殊的条件：`BEGIN` 和 `END`。
*   `BEGIN { 动作 }`：在**读取文件之前**执行。通常用来定义变量、打印表格的表头。
*   `END { 动作 }`：在**读取完文件所有内容之后**执行。通常用来打印总结信息、统计总数或平均值。

#### 示例：给表格加上表头和结尾总结
```bash
awk 'BEGIN {print "--- 成绩单开始 ---"} {print $1, $2} END {print "--- 成绩单结束 ---"}' score.txt
```
**输出**：
```text
--- 成绩单开始 ---
Tom 90
Jerry 88
Spike 60
--- 成绩单结束 ---
```

### 六、 实用高频范例（拿来即用）

在实际工作中，`awk` 的威力通常体现在以下这些场景中：

#### 1. 统计数据求和与平均值
利用 `awk` 可以在命令行中直接进行数学计算。
*   **计算第二列（数学成绩）的总分**：
    ```bash
    awk '{sum += $2} END {print "总分: " sum}' score.txt
    ```
    *(原理：每一行都把第二列的值累加到 `sum` 变量中，最后 `END` 模块打印出 `sum` 的最终值)*

*   **计算第二列的平均分**：
    ```bash
    awk '{sum += $2} END {print "平均分: " sum/NR}' score.txt
    ```

#### 2. 提取并过滤系统 IP 地址
通过过滤 `ifconfig` 的输出，快速抓取主机的 IP。
```bash
ifconfig eth0 | awk '/inet / {print $2}'
```

#### 3. 统计日志中某个接口或 IP 的出现次数
结合 `sort` 和 `uniq`，用 `awk` 提取日志中的特定列。
*   **示例**：统计 Nginx 访问日志 `access.log` 中，每个 IP 访问了多少次，并按次数排序。
    ```bash
    awk '{print $1}' access.log | sort | uniq -c | sort -rn
    ```

#### 4. 过滤掉文件的注释行和空白行
在查看配置文件时，常常需要排除 `#` 开头的注释和空白行。
```bash
awk '!/^#/ && !/^$/ {print $0}' nginx.conf
```

### 七、 总结：`awk` 常用组合速查表

| 功能场景 | 典型命令结构 |
| :--- | :--- |
| **提取指定列（如第1和第3列）** | `awk '{print $1, $3}' file` |
| **按特定分隔符切分** | `awk -F ',' '{print $1}' file` |
| **打印总行数** | `awk 'END {print NR}' file` |
| **查找第3列包含特定内容的行** | `awk '$3 ~ /keyword/ {print $0}' file` |
| **累加某列数值** | `awk '{sum += $1} END {print sum}' file` |

与 `grep` 的精准搜索不同，`awk` 更擅长“整理、重组和统计”数据。掌握了基础的列定位（`$1`, `$2`）与内置变量（`NR`, `NF`），您就已经能够应对日常工作中 90% 以上的文本排版与统计需求了。
