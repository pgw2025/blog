在前面几章中，我们已经搭建好了变量和数据类型这些“原材料”。今天，我们要让这些静态的数据“动起来”。

工业控制的本质，就是**数据处理和逻辑决策**。无论是计算一台水泵的运行效率、平滑地调节一个阀门的开度，还是判断安全光幕是否被触发，其底层都离不开数学运算（算术、求余）和逻辑运算（与、或、非、异或、位移）。

很多刚从梯形图（LAD）转过来的徒弟，一看到 SCL 的算术式和长串的布尔逻辑，就会觉得头大。但其实，SCL 在处理数学和逻辑运算时，有着传统图形化语言无法比拟的优势：**它完全遵循我们中学的代数书写习惯，计算链极短，且运行效率极高。**

今天，师父带你逐一拆解 SCL 的运算规则，并分享在复杂的工业物理环境中，如何写出防御性极强、绝不崩溃的代码。

## 1. 算术运算的基础与底层逻辑

在 SCL 中，基础算术运算包括：加（`+`）、减（`-`）、乘（`*`）、除（`/`）以及求余（`MOD`）。

### 1.1 运算符优先级与结合性

SCL 完全遵循标准的代数优先级规则。当你写下一个复杂的算术式时，CPU 会按照以下优先级从高到低依次计算：

1.  **括号** `()`：拥有最高优先级，能打破一切规则。
2.  **单目运算符**：如负号 `-`（如 `-#rValue`）。
3.  **乘除与求余**：`*`、`/`、`MOD`。
4.  **加减**：`+`、`-`。

```scl
// 示例
#rResult := #rA + #rB * #rC / (#rD - #rE);
```
在上面这行代码中，计算顺序为：
1. 计算括号内部的 `#rD - #rE`。
2. 计算乘法 `#rB * #rC`。
3. 将乘法结果除以括号的减法结果。
4. 最后加上 `#rA`。

*师父的建议*：**在实际项目中，哪怕你对优先级倒背如流，也请慷慨地使用括号 `()`。** 括号不仅能彻底消除因编译器解析差异导致的隐蔽 Bug，还能极大地提高代码的可读性，让后期的维护人员一眼看清你的数学公式。

---

### 1.2 ⚠️ 致命的工业级灾难：除以零（Division by Zero）

在 PC 软件开发中，除以零可能只会导致弹出一个未捕获异常窗口。**但在工业现场，如果 PLC 的 CPU 执行了除以零的操作，CPU 会瞬间报出严重编程错误，并立即切换到 STOP 状态，导致整条生产线瞬间瘫痪！**

```scl
// ❌ 极其危险的自杀式写法
#rFlowRate := #rTotalVolume / #rTimeDelta; // 如果系统刚启动，TimeDelta 尚未累积仍为 0.0，CPU 瞬间停机！
```

作为合格的高级工程师，你的代码必须具备**防御性（Defensive Programming）**。在进行任何除法或求余运算之前，必须对分母进行零值检查。

```scl
//  防御性工业级写法
IF ABS(#rTimeDelta) > 1.0E-6 THEN  // 浮点数防零：不能直接用 = 0.0，要用极小值判定
    #rFlowRate := #rTotalVolume / #rTimeDelta;
ELSE
    #rFlowRate := 0.0; // 默认分母为零时的安全输出
END_IF;
```

### 1.3 MOD（取余 / 模运算）的工业妙用

`MOD` 运算符用于计算两个整数相除后的余数。例如：`10 MOD 3` 的结果是 `1`。

在工业现场，`MOD` 是处理**循环运动、工位轮转、步序自增**的神器。

#### 典型场景：多工位旋转转盘（Carousel）
假设有一个 8 工位的旋转喷涂台，转盘每旋转一个工位，工位计数器 `#iStation` 就会自增 1。我们希望计数器在 `1` 到 `8` 之间无限循环：

```scl
// 每次旋转完成，触发自增
IF #bRotateCompleted THEN
    // 使用 MOD 约束范围在 0..7，再加 1 变换为 1..8
    #iStation := (#iStation MOD 8) + 1;
END_IF;
```
这种写法优雅地避免了繁琐的 `IF #iStation > 8 THEN #iStation := 1; END_IF;` 判定，且在数学上具有极高的严密性。

## 2. 比较运算：给 PLC 装上“量角器”

比较运算符包括：等号（`=`）、不等号（`<>`）、大于（`>`）、小于（`<`）、大于等于（`>=`）、小于等于（`<=`）。比较运算的结果是一个 **BOOL** 值（`TRUE` 或 `FALSE`）。

### 2.1 ⚠️ 浮点数（REAL）相等的惊天骗局

徒弟，听好了：**在任何 PLC 编程中，绝对不要直接用 `=` 或 `<>` 去比较两个浮点数（REAL / LREAL）！**

由于 IEEE 754 标准在二进制中无法精确表达大部分十进制小数（例如 $0.1$ 在二进制里是一个无限循环小数），浮点数在运算过程中会产生极其微弱的舍入误差。

```scl
// ❌ 永远无法触发的警报
#rValue1 := 1.0 - 0.9; // 在 CPU 中，rValue1 实际可能是 0.099999905
#rValue2 := 0.1;

IF #rValue1 = #rValue2 THEN
    #bAlarm := TRUE; // 这一行在 99% 的情况下永远不会被执行！
END_IF;
```

#### 正确的做法：引入“容差值”（Epsilon）
我们不判断两个浮点数是否绝对相等，而是判断它们的**绝对差值是否小于一个极小的可接受范围**。

```scl
//  正确的浮点数比较
IF ABS(#rValue1 - #rValue2) < 0.00001 THEN // 0.00001 为设定的容差
    #bAlarm := TRUE; // 差值极小，判定为“数学相等”
END_IF;
```

## 3. 逻辑运算：工业联锁的灵魂

逻辑运算符包括：与（`AND`）、或（`OR`）、非（`NOT`）和异或（`XOR`）。

### 3.1 异或（XOR）的工业魅力

很多初学者只知道 `AND` 和 `OR`，把 `XOR`（异或：两个输入不相同时输出 `TRUE`，相同时输出 `FALSE`）束之高阁。其实，`XOR` 在安全防错和状态翻转中有着极其优雅的应用。

#### 1) 物理按钮的“单按钮启停”
利用 `XOR` 运算，我们只需一行代码，就可以实现按一下按钮启动电机、再按一下停止电机的控制：

```scl
// 当按钮按下上升沿触发时，翻转电机运行状态
IF #bButtonTrigger_FP THEN
    #bMotorActive := #bMotorActive XOR TRUE; // 状态自翻转
END_IF;
```

#### 2) 双通道安全输入不一致性监控（Discrepancy Monitor）
在工业安全标准中，紧急停止按钮通常采用双通道（1路常开，1路常闭；或者双路常闭）。如果两个通道的状态不一致（一个通，一个断），说明传感器损坏或接线断路。
我们可以用 `XOR` 轻松诊断这个硬件故障：

```scl
// 正常状态下双常闭通道应完全相同。若不同，则触发不一致故障
#bDiscrepancyError := #bSafetyChannel1 XOR #bSafetyChannel2;
```

---

### 3.2 逻辑短路求值（Short-Circuit Evaluation）

在博途 S7-1200/1500 的 SCL 编译器中，逻辑表达式通常支持**短路求值**（编译器会自动进行效率优化）。

```scl
IF #bSensorInstalled AND (#arrBuffer[#iIndex] > 10.0) THEN
```
如果传感器未安装（即 `#bSensorInstalled` 为 `FALSE`），因为是 `AND` 关系，整行式子的结果注定是 `FALSE`。CPU 就会非常聪明地**不再去读取和评估后面的数组 `#arrBuffer[#iIndex]`**。
这不仅加快了扫描周期，更重要的是，它能**防止数组下标越界导致的崩溃**（如果 `#iIndex` 此时不合法，后半句的读取会导致 CPU 停机，而短路求值机制在前半句就将其截断，保护了系统的安全）。

## 4. 位操作：精细控制到每一个 Bit

对于总线通信（如 Profinet、Modbus）、变频器控制，我们必须具备直接操控 16 位或 32 位二进制数据的能力。SCL 提供了以下标准的移位指令：

*   **`SHL` (Shift Left)**：左移。右侧补 0。
*   **`SHR` (Shift Right)**：右移。左侧补 0（对于无符号数）。
*   **`ROL` (Rotate Left)**：循环左移。移出的高位重新补到低位。
*   **`ROR` (Rotate Right)**：循环右移。

```
 原始数据:       1 0 1 1  0 0 1 1  (16#B3)
 
 SHL(..., N:=2): 1 1 0 0  1 1 0 0  (16#CC, 左移2位，右侧补0)
 ROL(..., N:=2): 1 1 0 0  1 1 1 0  (16#CE, 循环左移2位，移出的高位10补到右侧)
```

### 4.1 位掩码（Bit Masking）的高级应用

假设你要控制一台步进驱动器。驱动器反馈回来的状态字是一个 `WORD` 类型的变量 `#wStatusWord`。
*   第 3 位代表：驱动器准备就绪（Ready）
*   第 7 位代表：驱动器故障报错（Fault）

我们该如何在 SCL 中优雅地提取这些状态？

```scl
// 1. 使用按位与 (AND) 与二进制掩码进行提取
// 2#0000_0000_0000_1000 对应十六进制 16#0008 (第3位为1)
#bReady := (#wStatusWord AND 16#0008) <> 0; 

// 2#0000_0000_1000_0000 对应十六进制 16#0080 (第7位为1)
#bFault := (#wStatusWord AND 16#0080) <> 0;
```
*师父的科普*：在 TIA 博途中，虽然我们可以使用前面学过的 `.X3` 和 `.X7` 这种 Slice 寻址，但位掩码（AND/OR）作为 IEC 61131-3 标准语法，在跨平台移植（比如移植到 Beckhoff 或者是 CodeSys 平台）时具有更好的通用性。

### 案例 1：Pt100 RTD 温度传感器标定与断线诊断

**场景描述**：
你正在开发一个塑料挤出机控制系统。温控模块接入了多路 Pt100 热电阻传感器。
*   温控模块（非标准 AI 通道）读取的原始整数值：温度每上升 0.1°C，读数值自增 1（即：寄存器读数为 `505` 代表 `50.5`°C）。
*   **安全要求**：当传感器接线断路时，模块的物理通道会输出最大饱和值 `32767`。程序必须立刻诊断出“传感器断线（Wire Break）”，切断加热器输出，防止设备因无节制加热而熔毁，同时输出摄氏度（°C）和绝对温度开尔文（K）。

```scl
FUNCTION "FC_Pt100_Scaling" : Void
{ S7_Optimized_Access := 'TRUE' }
VERSION : 0.1
   VAR_INPUT 
      iRawInput : Int;       // 模块读取的通道原始整数值 (1位小数)
   END_VAR

   VAR_OUTPUT 
      rTempCelsius : Real;   // 标定后的摄氏度输出 (°C)
      rTempKelvin : Real;    // 标定后的开尔文绝对温度 (K)
      bWireBreak : Bool;     // 传感器断线故障报警
   END_VAR

   VAR_TEMP 
      rTempCelsiusTemp : Real; // 内部临时摄氏度
   END_VAR


BEGIN
	// ==========================================
	// 1. 传感器物理故障安全诊断
	// ==========================================
	// 32767 是标准的断线溢出值，有些模块下溢为 -32768
	IF #iRawInput >= 32760 OR #iRawInput <= -32760 THEN
	    #bWireBreak := TRUE;
	    #rTempCelsius := -999.9; // 故障状态下输出极不正常的特殊值，提醒操作员
	    #rTempKelvin := 0.0;
	    RETURN; // 发生故障，立即截断执行，确保后续计算不会被执行
	ELSE
	    #bWireBreak := FALSE;
	END_IF;
	
	// ==========================================
	// 2. 物理量标定计算
	// ==========================================
	// 先将整型强转为实数，再除以 10.0，还原出一位小数的真实摄氏度
	#rTempCelsiusTemp := INT_TO_REAL(#iRawInput) / 10.0;
	
	// ==========================================
	// 3. 绝对温度转换与输出
	// 公式：K = °C + 273.15
	// ==========================================
	#rTempCelsius := #rTempCelsiusTemp;
	#rTempKelvin := #rTempCelsiusTemp + 273.15;
	
END_FUNCTION
```

### 案例 2：旋转编码器（HSC）皮带速度计算与断电溢出处理

**场景描述**：
输送皮带上安装了一个 24V 增量式编码器，接入 S7-1200 的高速计数器（HSC）通道中。
*   我们需要在固定的循环中断（如 OB30，100ms 周期精确触发）中，读取当前编码器的计数值 `#diCurrentCount`。
*   **物理参数**：编码器分辨率为 `1024` 脉冲/圈（PPR），传动轮直径为 `100.0` mm。
*   **难点（回零溢出）**：高速计数器是一个 32 位的带符号 DINT，范围为 `-2147483648` 到 `2147483647`。当计数器增加到最大值后，会自动回零翻转（例如：从 `2147483647` 变为 `-2147483648`）。直接相减会导致计算出极大的错误速度，必须通过算术逻辑进行修正。

```scl
FUNCTION_BLOCK "FB_SpeedCalculator"
{ S7_Optimized_Access := 'TRUE' }
VERSION : 0.1
   VAR_INPUT 
      diCurrentCount : DInt;      // 当前周期的高速计数器计数值 (100ms周期读取)
      rWheelDiameter : Real := 100.0; // 物理轮径 (mm)
      iPPR : Int := 1024;         // 编码器每圈脉冲数
      tCycleTime : Time := T#100ms; // 严格的采样计算周期
   END_VAR

   VAR_OUTPUT 
      rLinearSpeed : Real;        // 皮带线速度 (m/min)
   END_VAR

   VAR 
      diPrevCount : DInt;         // 静态变量：保存上一次的计数值
   END_VAR

   VAR_TEMP 
      diDeltaCount : DInt;        // 两个周期之间的脉冲差值
      rCircumference : Real;      // 物理轮周长 (m)
      rRPS : Real;                // 旋转转速 (圈/秒)
      rCycleSeconds : Real;       // 计算周期转换成的秒数
   END_VAR


BEGIN
	// ==========================================
	// 1. 物理常数计算 (轮子周长 mm 转换为 m)
	// 周长 C = π * D / 1000.0
	// ==========================================
	#rCircumference := (3.1415926 * #rWheelDiameter) / 1000.0;
	
	// ==========================================
	// 2. 周期时间转换 (TIME 转换为以秒为单位的 REAL)
	// 防御性设计：确保采样周期不为 0
	// ==========================================
	IF TIME_TO_DINT(#tCycleTime) > 0 THEN
	    #rCycleSeconds := DINT_TO_REAL(TIME_TO_DINT(#tCycleTime)) / 1000.0;
	ELSE
	    #rCycleSeconds := 0.1; // 默认防零崩溃备用时间（0.1s）
	END_IF;
	
	// ==========================================
	// 3. 脉冲差值计算与 32位 溢出自动纠偏
	// S7-1200/1500 在处理带符号 DINT 减法溢出时，
	// 如果利用 PLC 底层的无符号数自然溢出，可以得到极其简化的算法。
	// ==========================================
	#diDeltaCount := #diCurrentCount - #diPrevCount;
	
	// 针对极端的翻转边缘进行数学纠偏 (如果脉冲跨越了 2147483647 突变)
	// 如果由于倒转导致大跨度负值，或者正转导致突变，进行硬件范围重组
	IF #diDeltaCount < -10000000 THEN
	    // 正向溢出纠偏：由于计数器正向跨界翻转，相减得到了极大的负数
	    #diDeltaCount := #diDeltaCount + 2147483647 + 2147483648; 
	ELSIF #diDeltaCount > 10000000 THEN
	    // 反向溢出纠偏
	    #diDeltaCount := #diDeltaCount - 2147483647 - 2147483648;
	END_IF;
	
	// ==========================================
	// 4. 速度计算
	// 转速 (RPS) = 差值脉冲 / (分辨率 * 周期秒数)
	// 线速度 (m/min) = RPS * 周长 * 60.0
	// ==========================================
	IF #iPPR > 0 THEN
	    #rRPS := DINT_TO_REAL(#diDeltaCount) / (INT_TO_REAL(#iPPR) * #rCycleSeconds);
	    #rLinearSpeed := #rRPS * #rCircumference * 60.0;
	ELSE
	    #rLinearSpeed := 0.0;
	END_IF;
	
	// ==========================================
	// 5. 存储当前值供下一个周期使用
	// ==========================================
	#diPrevCount := #diCurrentCount;
	
END_FUNCTION_BLOCK
```

### 案例 3：物料储罐压力比例控制（P 控制器）与抗积分饱和上限限幅

**场景描述**：
我们需要开发一个简易的比例控制算法（P控制器，属于经典 PID 的核心部分），通过调节气阀的开度，使储罐压力控制在设定值（SetPoint）附近。
*   **计算公式**：$Output = K_p \times Error + Bias$
    其中 $Error = SetPoint - ProcessValue$。
*   **抗饱和限幅要求**：气阀物理开度限制在 `0.0% ~ 100.0%`。如果计算出的输出值超出了这个范围，必须进行**硬限幅（Clamping）**，防止气阀物理执行机构出现超限卡死或计算数值溢出。

```scl
FUNCTION "FC_ProportionalControl" : Real
{ S7_Optimized_Access := 'TRUE' }
VERSION : 0.1
   VAR_INPUT 
      rSetPoint : Real;       // 压力设定值 (SP, bar)
      rProcessValue : Real;   // 实际反馈压力 (PV, bar)
      rKp : Real;             // 比例增益 (Proportional Gain)
      rBias : Real;           // 控制偏置 (手动基础开度，%)
   END_VAR

   VAR_OUTPUT 
      bLimitActive : Bool;    // 阀门开度到达极限（限幅激活标志）
   END_VAR

   VAR_TEMP 
      rError : Real;          // 偏差值
      rCalcOutput : Real;     // 临时计算的未限幅输出
   END_VAR


BEGIN
	// ==========================================
	// 1. 偏差值计算
	// ==========================================
	#rError := #rSetPoint - #rProcessValue;
	
	// ==========================================
	// 2. 比例运算
	// ==========================================
	#rCalcOutput := (#rKp * #rError) + #rBias;
	
	// ==========================================
	// 3. 阀门开度硬限幅 (0.0% ~ 100.0%)
	// 防御性设计：确保输出不会突破物理机构边界
	// ==========================================
	IF #rCalcOutput > 100.0 THEN
	    #FC_ProportionalControl := 100.0; // 截断到最大值
	    #bLimitActive := TRUE;
	ELSIF #rCalcOutput < 0.0 THEN
	    #FC_ProportionalControl := 0.0;   // 截断到最小值
	    #bLimitActive := TRUE;
	ELSE
	    #FC_ProportionalControl := #rCalcOutput; // 在正常物理范围内
	    #bLimitActive := FALSE;
	END_IF;
	
END_FUNCTION
```

## 6. 常见错误与避坑指南 (Senior Tips)

### 6.1 错误一：用非等宽类型直接进行比较运算
```scl
// ❌ 极不推荐的跨阵营赋值比较
IF #wControlWord = #iStatusValue THEN ... // WORD 与 INT 的直接比较
```
*后果*：虽然在博途平台中，如果启用了隐式转换，编译器可能会放行。但在严格符合 IEC 标准的第三方系统或老版本西门子 CPU 中，这种写法会直接导致**编译报错**。
*解决方案*：进行强制类型转换，将两边的二进制宽度和类型对齐后再比较：
```scl
IF WORD_TO_INT(#wControlWord) = #iStatusValue THEN ...
```

### 6.2 错误二：除法操作中分子与分母“类型不统一”导致精度腰斩
```scl
#rResult := 1 / #rRealValue; // 假设 rRealValue 此时为 4.0
```
很多新手会认为计算结果是 `1 / 4.0 = 0.25`。
但在实际编译时，由于常数 `1` 被默认解析为 `INT` 类型，部分版本的编译器会尝试先将分母 `#rRealValue`（REAL）隐式转换为 `INT` 进行整数除法（`1 / 4` 商为 `0`），再把商 `0` 转换成浮点数，最终输出 `#rResult = 0.0`！
**黄金铁律**：**时刻保证算术式两端的数据类型一致。**
```scl
#rResult := 1.0 / #rRealValue; // 确保分子是 REAL
```
